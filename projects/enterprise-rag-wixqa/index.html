Enterprise Retrieval-Augmented Generation (RAG) with WixQA

GitHub Hugging Face Technical Report
We apply advanced techniques for building and evaluating Retrieval-Augmented Generation systems in enterprise environments.
  • Engineered an enterprise-scale RAG pipeline using the WixQA benchmark, processing the full training corpus of customer support queries and knowledge base articles.
  • Systematically optimized retrieval performance by benchmarking 16+ combinations of chunking strategies (300300300-500500500 tokens) and overlap configurations, identifying the optimal balance for Context Recall.
  • Architected a production-style stack utilizing PgVector with HNSW indexing, all-MiniLM-L6-v2 embeddings, and GPT-OSS-120B for high-fidelity response generation.
  • Quantified system accuracy through an LLM-as-a-Judge framework, achieving a significant Context Recall increase (0.40→0.810.40 \rightarrow 0.810.40→0.81) by tuning kkk-nearest neighbors and chunking parameters.
  • Validated generated output using F1 and ROUGE-1 metrics to ensure token-level overlap and content coverage across over a hundred thousand of synthetic baseline runs by using the hyper-accelerated Cerebras API for inference.
