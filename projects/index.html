M55M1 Elevator People Counting Industry Co-op with Nuvoton

We use the Nuvoton M55M1 and its Ethos-U55 NPU, the system to run optimized ML models directly on the device. We convert a top-down camera feed into a digital count without the latency or cost of cloud processing.
https://github.com/ai-ml-club-sjsu/m55m1-ElevatorCounting-YOLOv8n technical report Project slide deck (Google Slides) Mid semester update (Google Slides) Final semester update (Google Slides)
  • Optimized and deployed a real-time person-counting system on a resource-constrained Nuvoton platform, achieving 15 FPS within a strict 1.5MB SRAM footprint.
  • Engineered an INT8-quantized YOLOv8n pipeline using the Vera compiler and Arm Ethos-U55 NPU acceleration, leveraging Cortex-M55 Helium technology for efficient vector processing.
  • Curated a specialized dataset of 13,000+ images from Roboflow, and additionally using a 160° FOV camera in real-world elevator environments; published the augmented dataset to HuggingFace for community use.
  • Developed firmware using CMSIS-CSolution, integrating a custom Ultralytics training library to bridge the gap between high-level model training and low-level hardware execution.
  • Collaborated with Nuvoton hardware engineers to validate system performance for a live demonstration at the Sensors Converge conference.

Enterprise Retrieval-Augmented Generation (RAG) with WixQA

We apply advanced techniques for building and evaluating Retrieval-Augmented Generation systems in enterprise environments.
https://github.com/bencejdanko/wixqa-rag Hugging Face Dataset Technical Report
  • Engineered an enterprise-scale RAG pipeline using the WixQA benchmark, processing the full training corpus of customer support queries and knowledge base articles.
  • Systematically optimized retrieval performance by benchmarking 16+ combinations of chunking strategies (300300300-500500500 tokens) and overlap configurations, identifying the optimal balance for Context Recall.
  • Architected a production-style stack utilizing PgVector with HNSW indexing, all-MiniLM-L6-v2 embeddings, and GPT-OSS-120B for high-fidelity response generation.
  • Quantified system accuracy through an LLM-as-a-Judge framework, achieving a significant Context Recall increase (0.40→0.810.40 \rightarrow 0.810.40→0.81) by tuning kkk-nearest neighbors and chunking parameters.
  • Validated generated output using F1 and ROUGE-1 metrics to ensure token-level overlap and content coverage across over a hundred thousand of synthetic baseline runs by using the hyper-accelerated Cerebras API for inference.

Bert-TweetEval

BERT is an encoder-transformer based architecture that is more suitable for some tasks, such as classification. We benchmark both base and fine-tuned performance trained on the TweetEval dataset.
https://github.com/bencejdanko/bert-tweeteval Hugging Face Technical Report
  • Benchmarked and optimized lightweight Transformer architectures (DistilBERT, DistilRoBERTa) for multi-class emotion classification, achieving high-fidelity results (0.790.790.79 Accuracy) on imbalanced Twitter datasets.
  • Engineered a robust “Stress-Testing” framework to evaluate model resilience against distribution shifts, including character-level typos, emoji removal, and hashtag splitting.
  • Conducted comparative analysis between Encoder (BERT) and Decoder (Qwen-3, GPT-4o-mini) models, demonstrating that structured prompting allows 4B-parameter open-source models to rival closed-source SOTA performance.
  • Optimized model reliability by analyzing Expected Calibration Error (ECE) and implementing Macro-F1 early stopping to mitigate dataset imbalance and overconfidence.
  • Performed deep error analysis using token-level inspection to identify failures in subword tokenization (WordPiece/BPE) for misspelled emotional keywords, proposing Counterfactual Data Augmentation (CDA) as a solution.
